01
✦ Product analytics · B2B SaaS · SQL · Python

Trial Activation Analysis — Splendor Analytics

Defined what "Trial Activation" means for a B2B workforce management platform using 170,526 behavioural events from 966 organisations. Built a five-goal activation framework grounded in the product's core value loop — Create Shift → View Schedule → Assign Shift → Clock In/Out → Approve for Payroll — and constructed SQL staging and mart models to track activation at scale. Uncovered that 61% of organisations disappeared on Day 0, and that engagement volume alone does not predict conversion in B2B SaaS, pointing to external sales factors as the dominant driver.

Key findings
↗ 61% of orgs lost on Day 0 — an onboarding problem, not a conversion problem. Scheduling is the gateway module: zero organisations converted without using it. Activation rate: 26.6% · Conversion rate: 21.3%.

✦ Fraud analytics · Fintech · SQL In Progress

Account Takeover Investigation — Payflow Fintech

Investigating whether a coordinated account takeover campaign is underway at a fictional African fintech, Payflow. Working across three tables — user profiles, session logs, and transaction records — to detect anomalous login patterns, identify compromised accounts, segment affected users by risk tier and geography, and quantify the financial impact of flagged transactions. Signals being analysed include new device logins, ISP switching, failed login sequences, and cross-border transaction spikes.

Approach
↗ Synthetic African fintech dataset. Session-level anomaly detection combined with transaction flagging across user_profiles, user_sessions, and user_transactions — designed to mirror real fraud investigation workflows.
